## 数据存储与备案方案
1. 我的数据目前存放在哪里？  
> 我的数据目前分布在：  
> - 笔记本电脑本地：  
>   - `C:\Users\我的用户名\Documents` 里有工作文档  
>   - `D:\Projects` 里有代码和项目文件  
>   - 桌面上有临时文件和重要文档  
> - 手机：  
>   - 微信聊天记录和文件  
>   - 相册里的照片和视频  
> - 云端：  
>   - 坚果云同步了部分论文和PDF  
>   - GitHub 上存放了一些代码仓库  
>   - iCloud 同步了通讯录和备忘录  


2. 如果电脑坏了，你能恢复多少数据？  
> 如果电脑彻底坏了：  
> - 可以完整恢复：  
>   - GitHub 上的代码仓库  
>   - 坚果云里的PDF和部分文档  
>   - iCloud 同步的通讯录、备忘录  
> - 可能部分恢复：  
>   - 微信聊天记录（看是否开了聊天记录迁移/云备份）  
>   - 浏览器书签（如果登录了同步账号）  
> - 几乎无法恢复：  
>   - 只存在于桌面和 `D:\临时` 的一些文档和图片  
>   - 未同步的本地照片、视频  


3. 制定一个 3-2-1 备份方案  
- 主工作目录：  
  - Windows：`D:\Work`（所有文档、项目统一放这里）  
  - macOS：`~/Work` 或 `~/Documents/Work`  
- 本地备份（移动硬盘/U盘）：  
  - 设备：1 块移动硬盘（如 1TB）  
  - 方式：  
    - 每周一次，把 `D:\Work`/`~/Work` 整个复制到移动硬盘的 `Backup-Work` 目录  
    - 或使用备份软件（如 macOS 的 Time Machine / Windows 的文件历史记录 / FreeFileSync 等）自动同步  
- 云端备份（坚果云/iCloud/GitHub 等）：  
  - 文档类：  
    - 使用「坚果云」同步 `Work` 目录中的 `Documents` 子目录  
  - 代码类：  
    - 使用 Git，将代码仓库推送到 GitHub/Gitee  
  - 照片类（可选）：  
    - iCloud / Google Photos / 腾讯云相册等自动备份手机照片
