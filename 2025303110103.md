1. 我的数据目前存放在哪里？  
先把「数据」分成几类来想：

- 系统数据（操作系统、软件、配置）  
- 工作数据（文档、代码、照片、视频、项目文件等）  
- 账号数据（密码、书签、聊天记录、云端同步内容）

请你结合自己的实际情况，先在脑中点名一下：  
- 电脑（台式/笔记本）里有哪些重要文件？放在哪些文件夹？  
- 手机里有什么只保存在手机上的内容？  
- 有没有用过网盘/云盘（坚果云、iCloud、微信云备份、阿里云盘、百度网盘等）？  
- 代码有没有放在 GitHub / Gitee / GitLab？  
- 照片有没有在微信、QQ 相册、iCloud、Google Photos 之类的服务里？

然后把结果写成类似这样的一段话（这是示例，请改成你的真实情况）：

>   - `C:\Users\我的用户名\Documents` 里有工作文档

2. 如果电脑坏了，你能恢复多少数据？  

这里要先判断：  
- 哪些数据是只在电脑本地，有“唯一一份”？  
- 哪些数据同时在云端/移动硬盘/其他设备也有？

回答时可以分三类：

1）可以完整恢复的数据（有云同步 / 已经备份）  
2）可能部分恢复的数据（比如聊天记录、浏览器书签等，看是否开了同步）  
3）几乎无法恢复的数据（只在本机，从未备份）

示例（请改成你的真实情况）：

> 如果电脑彻底坏了：  
> - 可以完整恢复：  
>   - GitHub 上的代码仓库  
>   - 坚果云里的PDF和部分文档  
>   - iCloud 同步的通讯录、备忘录  
> - 可能部分恢复：  
>   - 微信聊天记录（看是否开了聊天记录迁移/云备份）  
>   - 浏览器书签（如果登录了同步账号）  
> - 几乎无法恢复：  
>   - 只存在于桌面和 `D:\临时` 的一些文档和图片  
>   - 未同步的本地照片、视频  

请你结合自己的实际情况写一段类似的评估。

3. 制定一个 3-2-1 备份方案  

3-2-1 原则：  
- 至少 3 份数据副本  
- 存在 2 种不同介质上（例如：电脑硬盘 + 移动硬盘 + 云端）  
- 至少 1 份在异地（通常就是云端）

下面给一个模板方案，你可以直接填写适合自己的路径/工具：

**示例方案 1（适合普通办公/学习）：**

- 主工作目录：  
  - Windows：`D:\Work`（所有文档、项目统一放这里）  
  - macOS：`~/Work` 或 `~/Documents/Work`  
- 本地备份（移动硬盘/U盘）：  
  - 设备：1 块移动硬盘（如 1TB）  
  - 方式：  
    - 每周一次，把 `D:\Work`/`~/Work` 整个复制到移动硬盘的 `Backup-Work` 目录  
    - 或使用备份软件（如 macOS 的 Time Machine / Windows 的文件历史记录 / FreeFileSync 等）自动同步  
- 云端备份（坚果云/iCloud/GitHub 等）：  
  - 文档类：  
    - 使用「坚果云」同步 `Work` 目录中的 `Documents` 子目录  
  - 代码类：  
    - 使用 Git，将代码仓库推送到 GitHub/Gitee  
  - 照片类（可选）：  
    - iCloud / Google Photos / 腾讯云相册等自动备份手机照片  

你可以照着这个格式，具体写出自己的 3-2-1 方案，例如：

> 3-2-1 备份方案：  
> • 主工作目录：  
>   - `D:\Work`（所有文档、项目统一放这里，不再散落在桌面和下载文件夹）  
> • 本地备份：  
>   - 1TB 移动硬盘，路径 `E:\Backup-Work`  
>   - 每周日晚上，用 FreeFileSync 将 `D:\Work` 同步到 `E:\Backup-Work`  
> • 云端备份：  
>   - 坚果云同步 `D:\Work\Docs`（文档）  
>   - GitHub 备份 `D:\Work\Code`（代码，使用 git 推送）  
>   - 重要证件扫描件单独放在坚果云的“重要文件”文件夹中
