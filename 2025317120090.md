##三、 定制化“3-2-1”备份蓝图
为了保护科研心血，我计划实施以下差异化存储方案：

生产环境：电脑本地 SSD 高速盘。

物理冷备份：外接 2T 加密移动硬盘。

云端热备份：GitHub（脚本类）+ 百度网盘/坚果云（论文与结果类）。

磁盘存储：存放 VCF、Fastq 等无法上传云端的大型基因文件。

分布式云存储：管理版本更迭频繁的 R、C++、Shell 脚本。

利用 GitHub 将核心分析流程（Pipeline）托管在云端，确保在实验室与宿舍之间可以无缝切换。

##四、 核心管理准则 (Data SOP)
命名禁令：坚决杜绝 final_v1.R, data_new.csv。统一采用 [YYYYMMDD]_[Project]_[Content].ext 格式。

Git 协同：大文件严禁 git add。通过 .gitignore 过滤 .vcf 和 .bam，仅记录数据处理的代码，不记录大数据。

环境对齐：每月导出一次 Conda 环境清单 (conda env export > environment.yml)。数据丢了可以重跑，但如果环境变了，实验将失去可重复性。

定期盘点：每两周手动同步一次 E 盘的 D2RS 核心目录到移动硬盘，并开启指纹校验。
