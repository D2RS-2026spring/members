# 数据安全与备份方案（3-2-1 备份策略）

## 1. 你的数据目前存放在哪里？
目前我的数据主要存放在个人电脑的本地硬盘中，包括课程作业、实验数据、文献资料以及论文相关文件。主要存放在电脑的工作文件夹中，例如：

- `D:\Research`
- `D:\Coursework`
- `D:\Literature`

这些文件夹用于分类存储科研数据、课程作业和文献资料。

---

## 2. 如果电脑坏了，你能恢复多少数据？
如果电脑突然损坏，而没有进行备份，我可能只能恢复少量已经上传到云端或发送过的文件（例如邮件附件或聊天记录中的文件）。  
大部分本地存储的实验数据、文献整理和课程作业可能无法恢复，数据丢失风险较大。

因此，需要建立系统化的数据备份方案。

---

## 3. 制定一个 3-2-1 备份方案

**3-2-1 备份原则：**
- **3** 份数据副本（1 个主数据 + 2 个备份）
- **2** 种不同存储介质
- **1** 份异地/云端备份

### 备份方案设计

- **主工作目录：**  
  `D:\Research`（电脑本地工作文件夹，用于日常科研与课程作业）

- **本地备份：**  
  移动硬盘（每周同步一次电脑数据，例如 `Research_Backup` 文件夹）

- **云端备份：**  
  坚果云 / iCloud / GitHub  
  - 文档、论文、代码 → GitHub  
  - 实验数据、报告 → 坚果云同步

---

## 总结

通过采用 **3-2-1 备份策略**：

- 本地电脑保存 **原始工作数据**
- 移动硬盘保存 **离线备份**
- 云端平台保存 **异地备份**

这样即使出现 **电脑损坏、硬盘损坏或误删除** 的情况，也可以最大程度地恢复数据，保障科研和学习数据的安全。
