### 第一部分：研究数据管理与 3-2-1 备份方案

**1. 你的数据目前存放在哪里？**
我的数据目前存放在搭载 macOS Tahoe 26 的电脑本地磁盘的项目文件中，同时每天会定时备份到指定的移动硬盘里。

**2. 如果电脑坏了，你能恢复多少数据？**
我能恢复截至前一天（即最后一次备份时）的所有原始数据、处理脚本和相关文件。唯一会丢失的是在最后一次备份之后（即当天）新编写的代码或数据处理产生的结果。目前的方案存在物理层面的单点故障风险：如果装有电脑和移动硬盘的背包同时遗失，或实验室遭遇突发灾害，数据将面临彻底丢失的风险。

**3. 制定一个 3-2-1 备份方案：**



* **主工作目录：** macOS 本地磁盘项目文件夹，路径为 `~/Documents/Research/Soil_Mapping_Project/`。该目录下严格划分子文件夹以符合可重复性研究规范：`data/raw/`（存放原始遥感影像与土壤数据）、`scripts/`（存放 Python 与 R 处理脚本）、`outputs/`（存放模型输出结果）以及 `manuscript/`（存放正在返修的论文手稿）。
* **本地备份：** 指定的移动硬盘，路径为 `/Volumes/External_HDD/Research_Backup/`。每天定时将主工作目录下的所有变动增量同步至该外接存储路径。
* **云端备份：** 百度网盘与 GitHub 双线异地备份。大体积的空间数据集、模型输出结果及论文文档通过百度网盘客户端自动同步，云端路径为 `/我的网盘/工作空间/Soil_Mapping_Project_Cloud/`；所有的 Python 和 R 处理代码则推送至 GitHub 的远端私有仓库，在实现异地存储的同时进行代码的版本控制。
