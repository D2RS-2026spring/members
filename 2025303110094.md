\## 3-2-1保存策略



严格遵循 3-2-1 备份原则（即保留 3 份数据副本、使用 2 种不同存储介质、存放 1 份异地备份），具体执行方案如下：



\*\*3 份数据副本：\*\*

\* \*\*副本一（主工作副本）：\*\* 存放于 Windows 台式机的工作目录中，用于日常读取、修改与高强度的数据处理。

\* \*\*副本二（本地物理备份）：\*\* 定期（如每周六）将工作目录下的重要新增数据（尤其是不可复现的实验结果和精修照片）手动复制到大容量的外接移动硬盘中。

\* \*\*副本三（云端异地备份）：\*\* 通过自动化同步工具和版本控制系统，将轻量级但极具价值的数据推送到云端服务器。



\*\*2 种不同存储介质：\*\*

\* \*\*介质 A：\*\* 电脑内置的高速 NVMe 固态硬盘（提供极速的读写性能）。

\* \*\*介质 B：\*\* 外接的移动硬盘（物理层面独立于电脑主机，防范系统崩溃、断电损坏或勒索病毒感染）。



\*\*1 份异地备份（云端分布）：\*\*

\* \*\*代码与纯文本报告：\*\* 通过 Git 进行版本控制，并推送到 GitHub 远程仓库进行异地托管。

\* \*\*文档与实验表格：\*\* 利用坚果云或 iCloud 进行实时自动同步。这不仅实现了异地容灾，还确保了在台式机、Alienware 笔记本以及 iPad Pro 之间能够无缝切换并查阅最新进度的文献与论文。
