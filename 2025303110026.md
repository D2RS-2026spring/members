## 我的数据存储与备份策略

### 1. 我的数据存放位置

目前我的数据主要存储在 Windows 电脑本地的项目文件夹中，例如：C:\Users\86132\Documents\projects
每个项目会单独建立文件夹，并按照数据分析项目结构进行组织，例如：
project/
data/
scripts/
results/
report/

这种结构可以保持项目文件清晰，方便管理和查找。

---

### 2. 如果电脑损坏可以恢复多少数据

如果电脑发生损坏：

- GitHub 上保存的代码和文档可以恢复
- 云盘同步的数据可以恢复
- 本地未备份的数据可能会丢失

预计可以恢复 **约 80% 的重要数据**。

---

### 3. 我的 3-2-1 备份策略

**主工作目录（Primary Working Directory）**

电脑本地工作目录：
C:\Users\86132\Documents\projects

用于日常学习、课程作业和数据分析项目。

---

**本地备份（Local Backup）**

使用移动硬盘或 U 盘进行备份：

- 每周定期备份一次项目文件夹
- 重要文件额外保存副本

---

**云端备份（Cloud Backup）**

使用云端服务进行同步：

- GitHub：保存代码和 Markdown 文档
- OneDrive 或坚果云：保存数据文件和重要资料

---

### 我的备份结构

本地电脑（主工作目录）
↓
移动硬盘 / U盘（本地备份）
↓
GitHub / 云盘（云端备份）

该方案符合 **3-2-1 数据备份原则**：

- **3 份数据副本**
- **2 种不同存储介质**
- **1 份异地备份**

这样可以有效保证数据安全，并在设备损坏时快速恢复数据。
