## 模型比较结果
1. **gpt-4o**：旗舰级多模态大模型，具备极快的响应速度和强大的视觉、文本跨模态理解能力，综合推理与代码编写表现顶尖，适合处理极其复杂的跨学科综合任务。
2. **gemini-pro-2.5**：拥有超大上下文窗口的优势，能够一次性吞吐和分析海量文献、代码库或长视频，非常适合大规模数据预处理和长文档信息提取。
3. **claude-sonnet-3-6**：在速度与智能之间取得了极佳的平衡，在代码生成、重构以及复杂逻辑推演方面表现极为出色，指令遵循能力极强，是理想的日常编程与科研助手。
4. **deepseek-r1**：顶级的强推理模型，尤其在数学计算、代码逻辑推理上达到极高水平，极具性价比，非常适合处理深度思考任务或作为复杂代码查错的逻辑引擎。
5. **kimi-2.5**：国内领先的长文本处理模型，本土化支持极佳，擅长快速解析大量中文长文档、PDF 论文或报告，是进行文献综述和快速信息检索的高效工具。

## 存储与备份策略
1. **数据当前存储位置**：
   - **实验与科研数据**：集中存放在 `E:\` 根目录下。E 盘被划分为纯数据盘，且已严格按照“不同实验项目”和“不同数据格式”建立了清晰的分类文件夹，数据管理较为规范。
   - **个人数据隐患**：目前存在局部数据单点故障风险。一旦电脑发生物理损坏，虽然可以通过实验室的 NAS 恢复核心实验数据，但由于 NAS 未涵盖其他个人数据（如学习笔记、生活文档等），这部分资料将面临永久丢失的风险。

2. **3-2-1备份方案**：
   - **主工作目录**：维持 `E:\` 根目录的分类体系，确保原始实验数据、代码脚本与个人文档各自在独立的子文件夹中，日常读写均在此主目录下完成。
   - **本地备份**：以实验室 NAS 作为主要的本地备份节点。不仅限于实验数据，每周需定期将 E 盘中的个人核心资料一并同步至 NAS，构成完整数据的第 2 份本地副本，确保设备损坏时能通过局域网快速恢复全部工作与生活环境。
   - **云端备份**：
     ① 代码与文档：全部项目代码与轻量级 Markdown 笔记通过 Git 进行版本控制，完成阶段性任务后执行 `git commit && git push` 同步至 GitHub 仓库。
     ② 数据集与大文件：针对 NAS 无法覆盖或需要极高安全性的重要个人资料、环境配置备份及大型数据集，定期打包压缩并上传至百度网盘，形成第 3 份异地容灾副本，彻底补全备份短板。
