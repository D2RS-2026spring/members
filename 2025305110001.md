\#  二、存储备份策略

\##  1. 数据当前存放位置

目前核心数据主要存放在：

-笔记本电脑内置 SSD：存放全部科研项目文件（实验数据、分析脚本、报告草稿等）

-手机本地 / 微信收藏：同步少量常用文档、会议记录

-GitHub 公开 / 私有仓库：托管代码类数据（数据分析脚本、可视化代码）

\## 2. 电脑损坏后可恢复的数据量

如果电脑硬件彻底损坏（如 SSD 故障）：

仅能恢复约 25% 的数据：包括同步到 GitHub 的代码、手机收藏的零散文档，以及部分自动同步到 iCloud 的笔记

约 75% 的核心实验原始数据、未提交的代码版本、本地报告草稿会完全丢失



\## 3. 3-2-1 备份方案（贴合生态学科研场景）

遵循「3 份副本、2 种介质、1 份异地」的 3-2-1 原则，方案如下：

\###主工作目录：

/Users/ecology-msc/projects/soil-ecology-2026/（Mac）

或 C:\\Users\\ecology-msc\\projects\\soil-ecology-2026\\（Windows）

内部按规范路径分类：data/raw/（原始数据）、data/processed/（处理后数据）、scripts/（分析代码）、reports/（论文 / 报告）
