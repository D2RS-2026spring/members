## 文件存储与备份策略（作业1）
### 1. 当前数据存放位置
- 课程数据、代码、文档：存放在电脑本地硬盘
- 课程相关代码：已同步至 GitHub 课程仓库
- 临时文件、截图等：散落在桌面与下载文件夹

### 2. 风险评估
如果电脑损坏、丢失或系统崩溃：
- GitHub 上的代码与文档可 100% 恢复
- 本地未备份的原始数据、图表、报告等文件会全部丢失

### 3. 3-2-1 备份方案
- 主工作目录：电脑本地硬盘（日常编写代码、处理数据）
- 本地备份：移动硬盘 / U 盘，每周手动同步一次
- 云端备份：坚果云，设置自动同步，支持多设备访问
- 异地备份：GitHub 远程仓库，用于代码与文档版本管理与异地存储

### 4. 文件管理规范
- 文件名：全英文、有意义、不使用中文与空格
- 文件路径：统一使用相对路径，避免绝对路径报错
- 原始数据：仅读取，不直接修改，所有处理步骤用代码记录
- 项目结构：按 data/、scripts/、results/、report/ 分类存放
