# 个人作业提交

## 第一部分：个人存储与备份策略 (3-2-1 原则)

### 1. 现状思考
* **我的数据目前存在哪里？**
  目前我的大部分学习资料、代码和实验数据都存放在个人笔记本电脑的本地硬盘 (C盘和D盘) 中，少量文件通过微信或 QQ 传输遗留在缓存文件夹中。
* **如果电脑坏了，我能恢复多少数据？**
  如果电脑突然损坏且硬盘无法读取，我只能恢复之前碰巧上传到 GitHub 的少量代码，以及通过社交软件发送过的部分文档。最近几周的本地工作进度将会全部丢失，数据恢复率极低。

### 2. 制定 3-2-1 备份方案
为了确保数据安全，我规划了以下符合“3份副本、2种介质、1份异地”原则的备份方案：

| 位置 | 用途 | 存储介质/工具 | 备份频率 |
| :--- | :--- | :--- | :--- |
| **主工作目录** | 日常代码编写与数据处理 | 笔记本电脑本地 SSD | 实时 |
| **本地备份** | 预防电脑系统崩溃或硬件损坏 | 1TB 移动硬盘 | 每周手动备份一次 |
| **云端备份** | 预防设备丢失、火灾等物理级灾难 | 坚果云 (自动同步) + GitHub (代码版本控制) | 坚果云实时同步，Git 每日 Push |

## 第二部分：大模型比较与评估

**测试问题：** “如何设计一个可重复的微生物组数据分析流程？”

**参与测试的 5 个大模型：** GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro, Kimi (月之暗面), 文心一言 4.0

### 模型回答差异比较：

1. **GPT-4o:** 回答结构极为严谨。不仅给出了从质控、拼接、比对到多样性分析的完整宏基因组/扩增子分析步骤，还重点强调了“可重复性”的核心：强烈推荐使用流程管理工具（如 Snakemake, Nextflow）和容器化技术（Docker, Singularity），并给出了具体的工具名称，专业度极高。
2. **Claude 3.5 Sonnet:** 回答详尽且逻辑清晰，排版最易于阅读。在强调容器化和版本控制（Git）的同时，补充了关于数据管理（如 FAIR 原则）和环境配置文件（如 conda `environment.yml`）的细节，对科研工作者非常实用。
3. **Gemini 1.5 Pro:** 能够准确覆盖微生物组分析的基础流程，但在“可重复设计”这一附加要求的深度上略逊于 GPT-4 和 Claude，更多是在罗列分析工具，对 workflow 框架的强调不够突出。
4. **Kimi (月之暗面):** 中文表现优秀，能够快速给出一个通用的生物信息学分析框架，提到了使用脚本自动化，但对于微生物组特有的分析细节（如 ASV/OTU 聚类工具 DADA2）和高级可重复性工具缺乏深入探讨。
5. **文心一言 4.0:** 回答较为宏观，虽然涵盖了数据收集、处理、分析的步骤，但整体偏向泛泛而谈，缺乏针对“微生物组”领域的专业术语和具体生信软件的推荐，且对“可重复性”的具体技术实现指导较弱。

### 综合表现排序 (从高到低)：

1. **Claude 3.5 Sonnet** (综合实力强，对生信流程的最佳实践理解深刻，排版极佳)
2. **GPT-4o** (技术深度最高，精准切中 Nextflow/Docker 等可重复性核心)
3. **Gemini 1.5 Pro** (流程完整，中规中矩)
4. **Kimi** (中文理解力好，框架清晰，但缺乏生信专业细节)
5. **文心一言 4.0** (过于宏观，缺乏具体工具和代码层面的指导)
