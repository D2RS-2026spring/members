## 模型比较结果

1. **GPT-4o**  
   - 回答最详细，逻辑清晰，能够深入解释复杂概念。  
   - 速度中等，适合需要高质量输出的任务。  
   - 准确性高，几乎无事实错误。

2. **Gemini Pro**  
   - 响应速度最快，适合快速问答。  
   - 回答简洁，但有时缺乏深度。  
   - 准确性尚可，但在专业领域偶有疏漏。

3. **DeepSeek-R1**  
   - 在技术问题上表现突出，代码生成能力强。  
   - 推理过程透明，适合学术研究辅助。  
   - 速度略慢，但结果可靠。

4. **Claude 3.5 Sonnet**  
   - 语言表达自然，适合文案创作。  
   - 对长上下文处理优秀，但中文支持稍弱。  
   - 综合表现均衡。

## 存储与备份策略

### 1. 我的数据目前存放在哪里？
- **本地主目录**：`/Users/lhua/projects/`  
  按项目组织，每个项目包含 `data/raw/`（原始数据只读）、`data/processed/`（清洗后数据）、`scripts/`（分析代码）、`results/`（输出图表）、`report/`（报告文稿）。文件名采用全英文、日期开头（例如 `2026-03-11_experiment1_raw.csv`），无空格。
- **实验室NAS**：`/Volumes/nas/lhua_data/`  
  用于团队共享的大型数据集，自动备份。
- **云端同步**：坚果云（`/Users/lhua/Nutstore/`）  
  同步关键文档、论文草稿和配置文件。

### 2. 如果电脑坏了，我能恢复多少数据？
- **当前恢复能力**：  
  - 坚果云每日自动同步约80%的活跃文档和代码。  
  - 外置移动硬盘每周完整备份一次，最多丢失一周内的修改。  
  - NAS每晚同步本地 `projects/` 目录，保留30天历史版本。  
  - 总体可恢复90%以上的数据，仅当天未同步的少量修改可能丢失。

### 3. 我的 3-2-1 备份方案
| 备份层级 | 存储位置与介质 | 说明 |
|----------|----------------|------|
| **主工作目录**（第1份） | 本地笔记本SSD<br>`/Users/lhua/projects/` | 日常读写工作区，按项目组织。 |
| **本地备份**（第2份） | 外置移动硬盘（2TB）<br>每周五执行 `rsync -av --delete` | 介质独立，防止电脑硬盘同时损坏。 |
| **异地备份**（第3份） | **坚果云**（自动同步）<br>**GitHub**（私有仓库托管代码）<br>**实验室NAS**（每晚定时备份） | 满足异地要求，防止火灾/盗窃；GitHub同时提供版本控制。 |

- **3份副本**：本地 + 移动硬盘 + 云端（坚果云 + GitHub + NAS）  
- **2种介质**：SSD + 机械硬盘 + 云存储  
- **1份异地**：坚果云/GitHub位于校外数据中心，NAS位于另一栋楼  

**执行细节**：
- 坚果云同步 `projects/` 下的 `report/`、`scripts/` 和部分 `processed/` 数据，忽略大文件（如原始测序数据）。
- GitHub托管所有代码和 `.qmd` 文档，每个项目单独仓库，定期 `git push`。
- NAS使用 `rsync` 拉取整个 `projects/`，保留30天快照。
