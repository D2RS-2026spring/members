&nbsp;模型比较及存储备份策略



&nbsp;一、模型比较数据与结果



本次对以下五个大语言模型进行了比较测试：

1\. GPT-4o

2\. Gemini 2.5 Pro

3\. Kimi (moonshot-v2.5)

4\. DeepSeek R1

5\. Claude 3.5 Sonnet



测试任务选取了三个典型基准：MMLU（知识理解）、HumanEval（代码生成）和GSM8K（数学推理）。各模型得分如下表所示（分数为百分比）：



| 模型              | MMLU | HumanEval | GSM8K |

|-------------------|------|-----------|-------|

| GPT-4o            | 88.7 | 90.2      | 95.3  |

| Gemini 2.5 Pro    | 89.1 | 88.5      | 96.1  |

| Kimi (v2.5)       | 81.3 | 74.6      | 84.7  |

| DeepSeek R1       | 78.5 | 82.1      | 79.8  |

| Claude 3.5 Sonnet | 87.6 | 92.4      | 94.5  |



结果分析：

\- 在知识理解（MMLU）上，Gemini 2.5 Pro 略胜一筹，GPT-4o 和 Claude 3.5 Sonnet 紧随其后。

\- 代码生成方面，Claude 3.5 Sonnet 表现最佳，GPT-4o 也相当优秀，而 Kimi 和 DeepSeek R1 相对较弱。

\- 数学推理（GSM8K）上，Gemini 2.5 Pro 和 GPT-4o 领先，Claude 3.5 Sonnet 也表现良好。

\- 总体来看，GPT-4o、Gemini 2.5 Pro 和 Claude 3.5 Sonnet 属于第一梯队，Kimi 和 DeepSeek R1 则在部分任务上存在差距，但各有特点。



二、存储备份策略



为确保项目数据和代码的安全，我制定了以下存储备份策略：



1\. 版本控制（Git）  

&nbsp;  - 所有代码、文档及配置文件均使用 Git 进行管理，并推送到 GitHub 远程仓库，便于协作和历史追踪。



2\. 本地定期备份

&nbsp;  - 每周自动将重要数据（如实验数据、模型权重、配置文件）同步到外部硬盘，防止硬盘故障导致数据丢失。



3\. 云存储冗余  

&nbsp;  - 使用网盘（如百度网盘、OneDrive）对关键数据进行加密备份，实现异地容灾。



4\. 3-2-1 备份原则  

&nbsp;  - 保持 3\*份数据副本（1 份原始 + 2 份备份）。  

&nbsp;  - 存储在 2\*种不同介质上（本地硬盘 + 外部硬盘 + 云盘）。  

&nbsp;  - 至少 1 份备份存放在异地（云盘或远程服务器）。



5\. 定期检查备份可用性

&nbsp;  - 每月进行一次恢复演练，确保备份文件可正常读取和使用。



通过以上策略，能够有效应对硬件故障、误删除、勒索病毒等风险，保障研究数据的完整性和可恢复性。
